# Implementation Skill - 계획서 기반 구현

이 skill은 작성된 명세서에 따라 구현을 진행합니다.

## 사전 조건
- 구현할 작업의 명세서가 `/specs/` 디렉토리에 존재해야 합니다
- 명세서 상태가 `⏳ 대기` 또는 `🔄 변경필요`여야 합니다

## 실행 단계

### 1. 명세서 확인
- 구현할 명세서 파일을 읽습니다
- 요구사항(REQ-X) 목록을 추출합니다
- 인터페이스 정의를 파악합니다
- 제약조건을 확인합니다

### 2. 추적성 매트릭스 업데이트 (구현 시작)
`/specs/TRACEABILITY.md`에서 해당 항목의 상태를 `🚧 진행중`으로 변경합니다.

### 3. 코드 구현
명세서의 각 요구사항을 순서대로 구현합니다:

```typescript
/**
 * [기능 설명]
 * @spec FEAT-XXX
 * @spec API-XXX (관련된 경우)
 */
```

구현 시 준수사항:
- 명세서의 인터페이스 정의를 정확히 따릅니다
- 제약조건(성능, 보안 등)을 충족합니다
- 코드에 @spec 주석으로 명세 ID를 참조합니다

### 4. 테스트 작성
검증 기준에 따른 테스트를 작성합니다:
- 단위 테스트: `/tests/[category]/[feature].test.ts`
- 통합 테스트: `/tests/integration/[feature].test.ts`

### 5. 추적성 매트릭스 업데이트 (구현 완료)
구현 및 테스트 완료 후 매트릭스를 업데이트합니다:
- 구현 파일 경로 기록
- 테스트 파일 경로 기록
- 상태를 `🧪 테스트중` 또는 `✅ 완료`로 변경

### 6. 명세서 체크리스트 업데이트
명세서 내 요구사항 체크리스트를 완료 처리합니다:
```markdown
- [x] REQ-1: [완료된 요구사항]
- [x] REQ-2: [완료된 요구사항]
```

## 구현 순서 가이드

### 패키지/모듈 구조
```
src/
├── [feature]/
│   ├── index.ts          # Public exports
│   ├── types.ts          # Type definitions
│   ├── [feature].ts      # Main implementation
│   └── [feature].test.ts # Unit tests
```

### 구현 체크리스트
- [ ] 타입 정의 완료
- [ ] 핵심 로직 구현
- [ ] 에러 처리 추가
- [ ] 단위 테스트 작성
- [ ] 통합 테스트 작성 (필요시)
- [ ] 린트 통과 확인
- [ ] 타입 체크 통과 확인

## 출력 형식

```
## 구현 완료 보고

### 명세서 정보
- **Spec ID**: FEAT-XXX
- **제목**: [명세서 제목]

### 구현 파일
- `src/[path]/[file].ts`
- `src/[path]/[file2].ts`

### 테스트 파일
- `tests/[path]/[file].test.ts`

### 요구사항 충족 현황
- [x] REQ-1: [설명]
- [x] REQ-2: [설명]

### 추적성 매트릭스 상태
- 이전: ⏳ 대기
- 현재: ✅ 완료

### 다음 단계
- [후속 작업 또는 의존 작업 정보]
```

## 주의사항
- 명세서에 없는 기능은 임의로 추가하지 않습니다
- 구현 중 명세 변경이 필요하면 먼저 명세서를 수정합니다
- 테스트 없이 구현 완료 처리하지 않습니다
- 모든 변경사항은 커밋 메시지에 Spec ID를 포함합니다
