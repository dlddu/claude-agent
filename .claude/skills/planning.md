# Planning Skill - 작업 분할 및 계획서 작성

이 skill은 앞으로 해야 할 작업을 분석하여 적절한 크기로 분할하고, 바로 다음에 수행할 작업에 대한 명세서를 작성합니다.

## 실행 단계

### 1. 현재 상태 파악
- `/specs/TRACEABILITY.md` 파일을 읽어 현재 프로젝트 상태를 파악합니다
- 기존 명세서들의 상태(⏳ 대기, 🚧 진행중, ✅ 완료 등)를 확인합니다
- 의존성 그래프를 분석하여 다음에 수행해야 할 작업을 식별합니다

### 2. 작업 분할
- 전체 작업을 논리적 단위로 분할합니다
- 각 단위는 1-4시간 내에 완료 가능한 크기로 설정합니다
- 의존성을 고려하여 작업 순서를 결정합니다

### 3. 다음 작업 선정
우선순위 기준:
1. 다른 작업에 의존성이 없는 작업
2. 상태가 `⏳ 대기`인 작업 중 우선순위가 높은 것
3. 구현 우선순위(Phase 순서) 준수

### 4. 명세서 작성
선정된 작업에 대해 적절한 명세서를 작성합니다:

**기능 명세서 (FEAT-XXX):**
- `/specs/features/FEAT-XXX.md` 경로에 생성
- 요구사항, 인터페이스, 제약조건, 검증 기준 포함

**API 명세서 (API-XXX):**
- `/specs/api/API-XXX.md` 경로에 생성
- 엔드포인트, Request/Response, Error Codes 포함

**인프라 명세서 (INFRA-XXX):**
- `/specs/infra/INFRA-XXX.md` 경로에 생성
- 구성 요소, 배포 전략, 환경 설정 포함

### 5. 추적성 매트릭스 업데이트
- `/specs/TRACEABILITY.md`에 새 항목 추가
- 상태를 `⏳ 대기`로 설정
- 의존성 그래프 업데이트 (필요시)

## 출력 형식

작업 완료 후 다음 정보를 보고합니다:

```
## 작업 분할 결과

### 전체 남은 작업
1. [작업 1] - 우선순위: High/Medium/Low
2. [작업 2] - 우선순위: High/Medium/Low
...

### 다음 수행 작업
- **Spec ID**: FEAT-XXX / API-XXX / INFRA-XXX
- **제목**: [작업 제목]
- **예상 범위**: [구현할 파일/기능 목록]
- **의존성**: [선행 작업 ID 또는 없음]

### 생성된 명세서
- 경로: /specs/[category]/[SPEC-ID].md
- 상태: ⏳ 대기
```

## 주의사항
- 명세서는 구현 전에 반드시 작성해야 합니다
- 기존 명세서와 ID가 중복되지 않도록 합니다
- 너무 큰 작업은 여러 명세서로 분할합니다
- 변경 이력을 명세서 하단에 기록합니다
